⚾ 野球スコアボード

0. 準備

大会情報（大会名、参加チーム名など）を設定します。

⚙️ 大会情報の設定
詳しい設定方法は こちら をご覧ください。
1. サーバーを起動

コマンドラインで以下を実行してください:

node server.js

サーバーはポート8080で起動します。本番環境では NODE_ENV=production node server.js を使用してください。

2. 基本的な使用フロー
  1. 操作パネルを開いて試合情報(チーム名、イニングなど)を設定
  2. 表示ボードを開いてスコアボードを表示
  3. OBSのブラウザソースに表示ボードのURLを設定
  4. 操作パネルで変更した内容が表示ボードにリアルタイム反映されます
3. 別のPCからアクセスする場合

サーバーを起動したPCのIPアドレスを確認します:

# Linux/Mac
hostname -I

# Windows
ipconfig

別のPCのブラウザで以下のようにアクセスします:

http://<サーバーのIPアドレス>:8080/operation.html
http://<サーバーのIPアドレス>:8080/board.html

例: http://192.168.1.100:8080/board.html

4. OBSでの設定方法
  1. OBSで「ソース」→「追加」→「ブラウザ」を選択
  2. URLに表示ボードのアドレスを入力 (例: http://192.168.1.100:8080/board.html)
  3. 幅: 1920px、高さ: 1080px（または配信解像度に合わせて調整）
  4. 「エフェクトフィルタ」→「クロマキー」を追加し、キーカラータイプで緑色を選択

操作パネルの初期値（大会名、チーム名など）は config/init_data.json ファイルで設定します。
自動生成ツール npm run init を使うと簡単に作成できます。

方法1: インタラクティブモード（推奨）

対話形式で質問に答えていくだけで設定ファイルを生成します:

npm run init
実行例:
大会名を入力してください [現在: 大会名]: 春季リーグ戦
試合の最終イニングを入力してください [現在: 9]: 9
参加チーム名を入力してください [入力終了: enterのみ]:
  チーム 1: 東京D
  チーム 2: 横浜S
  チーム 3: 大阪T
  チーム 4: (Enterで終了)
方法2: YAMLファイルから生成

設定ファイル（YAML形式）を作成して生成します:

ステップ1: サンプルファイルをコピー

cp config/config.yaml.example config/my-config.yaml

ステップ2: ファイルを編集

game_title: 春季リーグ戦
last_inning: 9
team_names:
  - 東京D
  - 横浜S
  - 大阪T

ステップ3: 設定ファイルから生成

npm run init config/my-config.yaml
方法3: コマンドライン引数で直接指定

一行のコマンドですべての情報を指定します:

npm run init -- -t "春季リーグ戦" -i 9 --teams "東京D,横浜S,大阪T"

オプション: -t 大会名、 -i 最終イニング(1-9)、 --teams 参加チーム(カンマ区切り)

自動生成ルール
  • 先攻チーム: 参加チームの1番目
  • 後攻チーム: 参加チームの2番目
  • イニング数: 1〜9回（試合前・試合終了を自動追加）
  • 参加チーム: 最低2チーム必要
💡 安心のバックアップ機能
既存の config/init_data.json がある場合、自動的に config/init_data.json.bak にバックアップされます。

⚠️ 複数人で操作する場合の注意
操作パネルを複数の端末で同時に開いた場合、最初に接続した人だけが操作可能（マスター）となり、 後から接続した人は閲覧のみ（スレーブ）となります。 詳細は「複数人での操作について」をご覧ください。
試合準備

1. 基本情報の設定

  • タイトル欄に大会名を入力する。
  • 先攻チーム・後攻チームをプルダウンから選択する。
  • 表示ボードに上記結果が反映されていることを確認する。
試合開始

1. イニングの進行

  • 攻守交代・出塁カードの 「進む」 を押す。
  • イニング表示が 「１回オモテ」 になったことを確認する。
  • その後は3アウトの度に 「進む」 を押し、「１回ウラ」→「２回オモテ」…「試合終了」となる。
  • もし、間違えてイニングを進めた場合、「戻る」 を押して修正する。

2. ピッチャーが投球した際の操作

ストライクの場合
  • Sの下の 「＋」 を押す。
  • 3振なら、「BSリセット」 を押し、Oの下の 「＋」 を押す。
ボールの場合
  • Bの下の 「＋」 を押す。
  • フォアボールになったら、「BSリセット」 を押し、攻守交代・出塁カード下段の 「1B」「2B」「3B」 を押す。
  • その際、既に出塁者がいた場合は、その出塁者も合わせて 「1B」「2B」「3B」 を押す。
ヒットの場合
  • 「BSリセット」 を押し、出塁先に合わせて攻守交代・出塁カード下段の 「1B」「2B」「3B」 を押す。
  • その際、既に出塁者がいた場合は、その出塁者も合わせて 「1B」「2B」「3B」 を押す。
得点が入った場合
  • スコアカードの当該チームの 「＋」 を押し、得点を入れる。
  • もし、得点を間違えた場合、「−」 を押して修正する。
ファールの場合
  • ストライク数に合わせて、Sの下の 「＋」 を押す。
アウトの場合
  • Oの下の 「＋」 を押す。
  • もし、アウト数を間違えた場合、「−」 を押して修正する。
💡 攻守交代のタイミング
3アウトになったら、攻守交代・出塁カードの 「進む」 を押す。
その結果、BSOカウント・出塁者がリセットされ、攻守が交代となる。
試合の初期化

攻守交代・出塁カードには2種類の初期化ボタンがあります：

方法1: 同じチームで次の試合を開始する場合

  • 攻守交代・出塁カードの 「🔄 試合初期化」 ボタンを押す。
  • 確認ダイアログが表示されるので、内容を確認して 「OK」 を押す。
  • イニング、得点、BSO・出塁者がリセットされ、試合前の状態に戻ります。
  • チーム名と大会名はそのまま維持されます。

方法2: 新しい大会を開始する場合

  • npm run init で新しい大会設定（config/init_data.json）を生成する。
  • 操作パネルの 「📋 新規大会で初期化」 ボタンを押す。
  • 確認ダイアログで 「OK」 を押す。
  • 新しい大会名、チーム名、イニング設定が読み込まれます。
⚠️ 注意事項
試合中でも初期化ボタンは押せますが、確認ダイアログで警告メッセージが表示されます。
初期化を実行すると、現在の試合状況はすべて失われますのでご注意ください。

マスター/スレーブシステムとは

このシステムでは、操作パネル（operation.html）を複数の端末から同時に開いた場合でも、 一度に1人だけが操作可能になるよう制御されています。

  • マスター（操作可能）：最初に接続した人が自動的にマスターになります。すべての操作が可能です。
  • スレーブ（閲覧専用）：2番目以降に接続した人は自動的にスレーブになります。画面を見ることはできますが、操作はできません。
自分の状態を確認する方法

操作パネルの画面上部（ナビゲーションバー右側）にステータスが表示されます：

👑 マスター (操作可能)
このバッジが表示されている場合、あなたが操作可能です。
👁️ スレーブ (閲覧専用)
このバッジが表示されている場合、閲覧のみ可能です。

スレーブの場合、画面上部に黄色の警告バナーが表示され、すべての操作ボタンが無効化（グレーアウト）されます。

推奨される使用方法
  • 1人を操作担当者として決める：試合開始前に、誰が操作パネルを操作するか決めておきましょう。
  • 他の人はモニター用として使う：スレーブ接続は、別の端末でリアルタイムの状況を確認するのに便利です。
  • 同じ端末で複数タブを開かない：無駄なスレーブ接続が増えるだけなので避けましょう。
マスター権限の解放

マスターが操作権限を他の人に譲りたい場合は、以下の方法があります：

  • 方法1：マスター解放ボタンを使う
    • 操作パネルの右側に表示される「👑 マスター権限」カード内の「🔓 マスター権限を解放」ボタンをクリック
    • 確認ダイアログで「OK」を押すと、次に古い接続（スレーブ）が自動的にマスターに昇格します
  • 方法2：ブラウザを閉じる/リロードする
    • マスターが操作パネルを閉じる、またはブラウザをリロードすると、自動的に次の人がマスターになります
よくある質問
Q: マスターがいきなり切断されたらどうなりますか？
A: 自動的に、次に早く接続したスレーブがマスターに昇格します。試合データは失われません。
Q: スレーブ状態で操作ボタンを押すとどうなりますか？
A: すべてのボタンが無効化されているため、押すことができません。
Q: 表示ボード（board.html）も同じ制限がありますか？
A: いいえ。表示ボードは常に閲覧専用で、何台でも同時に開くことができます。
Q: マスターがブラウザをリロードした場合は？
A: 一時的に他のスレーブがマスターに昇格しますが、元のマスターが再接続すると、再度マスターになることはありません。先着順のルールが適用されます。
Q: 誰もスレーブがいない状態でマスターが権限を解放したら？
A: 解放した人がそのままスレーブになります。次に誰かが接続したときに、その人がマスターになります。
💡 ヒント
複数人で作業する場合でも、基本的には1人が操作を担当し、 他の人はスレーブ接続で状況を監視する運用を推奨します。 これにより、操作ミスを防ぎ、スムーズな試合運営が可能になります。
